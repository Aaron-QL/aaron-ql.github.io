Docker网络

发表于 2020-08-29 | 分类于 知识点总结 ， 容器网络 |

网络驱动

  • bridge：默认的网络驱动。当应用程序需要在容器中独立运行时通常使用Bridge网络。
  • host：对于独立容器，删除容器和主机之间的网络隔离，让容器直接使用主机的网络。
  • overlay：覆盖网络将多个Docker守护进程连接在一起，使集群服务能够相互通信。
  • macvlan：允许为容器分配MAC地址，让它在网络中像一台物理设备。
  • none：禁用所有网络。

动态卷供应

发表于 2020-08-27 | 分类于 知识点总结 ， Kubernetes |

背景

我们目前使用的存储服务是MooseFS，部署工具会在启动k8s前为每个节点挂载好共享路径/mnt/shareroot。
当应用有存储需求时，需要手动在Pod中声明类型为hostpath的volume，并在.containers.volumeMounts中进行映射。

这种方案有以下弊端：

  • 需要为每个应用手动的在yaml中声明PV
  • 无法对卷的访问模式和容量进行限制及扩展

为此，我们引入 Kubernetes 中的PVC和StorageClass两个重要的资源对象来实现对存储的管理。

概念

PVC

PVC的全称是：PersistentVolumeClaim（持久化卷声明），PVC 是用户存储的一种声明，PVC 和 Pod 比较类似，Pod 消耗的是节点，PVC 消耗的是 PV 资源，Pod 可以请求 CPU 和内存，而 PVC 可以请求特定的存储空间和访问模式。

对于真正使用存储的用户不需要关心底层的存储实现细节，只需要直接使用 PVC 即可。但是通过 PVC 请求到一定的存储空间也很有可能不足以满足应用对于存储设备的各种需求，而且不同的应用程序对于存储性能的要求可能也不尽相同，比如读写速度、并发性能等。

k8s学习笔记-5 API约定

发表于 2020-07-11 | 分类于 学习笔记 ， Kubernetes |

本文翻译自API Conventions，面向想要深入理解k8s结构和用户以及想要扩展k8s API的开发者。在对象管理概述中可以找到将资源与kubectl一起使用的介绍。

K8S API的约定旨在简化客户端的开发，确保可以实施能够在各种用例中保持一致的配置机制。

K8S API通用的方式是RESTful——客户端通过标准的HTTP动词（POST、PUT、DELETE和GET）创建，更新，删除或检索一个对象的描述——并且这些API优先接受并返回JSON。k8s也为非标准动词开房了其他端点，并允许使用其他内容类型。所有服务器接受和返回的JSON都有一个结构，用“kind”和“apiVersion”字段标识。如果HTTP header头里有这样的字段，则应该在JSON里镜像一份内容，而不是只在header头里。

理解Pod

Pod是Kubernetes应用程序的基本单元，即它是Kubernetes对象模型中创建或部署的最小和最简单的单元。

Pod表示在集群上运行的进程。一个Pod中可以运行一个或多个进程。

每个Pod表示运行给定应用程序的单个实例。如果希望横向扩展应用程序则应使用多个Pod。

k8s学习笔记-2 Kubernetes对象

发表于 2020-06-14 | 分类于 学习笔记 ， Kubernetes |

Kubernetes对象

理解Kubernetes对象

在 Kubernetes 系统中，Kubernetes 对象 是持久化的实体。Kubernetes 使用这些实体去表示整个集群的状态。特别地，它们描述了如下信息：

  • 哪些容器化应用在运行（以及在哪个 Node 上）
  • 可以被应用使用的资源
  • 关于应用运行时表现的策略，比如重启策略、升级策略，以及容错策略
    Kubernetes 对象是 “目标性记录” —— 一旦创建对象，Kubernetes 系统将持续工作以确保对象存在。通过创建对象，本质上是在告知 Kubernetes 系统，所需要的集群工作负载看起来是什么样子的，这就是 Kubernetes 集群的 期望状态（Desired State）。

操作 Kubernetes 对象 —— 无论是创建、修改，或者删除 —— 需要使用 Kubernetes API。比如，当使用 kubectl 命令行接口时，CLI 会执行必要的 Kubernetes API 调用，也可以在程序中使用 客户端库 直接调用 Kubernetes API。

简介

Kubernetes是什么？

Kubernetes是一个可移植的、可扩展的开源平台，用于管理容器化的工作负载和服务，可促进声明式配置和自动化。Kubernetes 拥有一个庞大且快速增长的生态系统。Kubernetes 的服务、支持和工具广泛可用。

名称 Kubernetes 源于希腊语，意为 “舵手” 或 “飞行员”。Google 在 2014 年开源了 Kubernetes 项目。Kubernetes 建立在 Google 在大规模运行生产工作负载方面拥有十几年的经验的基础上，结合了社区中最好的想法和实践。
